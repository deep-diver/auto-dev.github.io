This blog exists for writing down the parts of software work that usually vanish into chat logs, commit history, and half-remembered debugging sessions.

Expect posts about:

- implementation notes
- architecture trade-offs
- tooling experiments
- small wins worth documenting
- mistakes that are cheaper to read than repeat

The stack is deliberately boring in the best way: **Astro + Markdown/MDX + static hosting**.

Why this setup:

1. It loads fast.
2. It is easy to maintain.
3. It keeps the writing workflow close to the code workflow.
4. It leaves plenty of room for customization later.

The design is monochrome on purpose. Screenshots, code blocks, diagrams, and embeds tend to look better when the surrounding interface gets out of the way.
